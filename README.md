
### 설계 철학

#### 확장성과 안정성

확장성 -> 전략 패턴, FeignClient와 Profile 어노테이션을 통해 새로운 AI 도입과 테스트 환경에 대해 기존의 코드 수정없이 진행시켰습니다.

안정성 -> Resilience4J를 통해 외부 서버에서 장애가 발생했을 경우 의미없는 지연 시간을 줄이고자 했습니다.  
시간만 더 있었다면 퍼사드에서 Rate Limiter와 세마포어를 도입해 보다 더 안정적인 동시성을 확보할 계획이었습니다.
